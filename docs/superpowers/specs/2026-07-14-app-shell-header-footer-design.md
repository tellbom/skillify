# Skillify 应用头部与页脚设计

## 目标

修正业务页面共享头部，使其视觉表现与 `web/temp/Skillify Platform Interface Design/Skillify.dc.html` 中的原型一致，并为所有业务页面增加与现有深色主题统一的完整页脚。

## 范围

- 仅修改业务路由使用的共享应用壳，不改变登录页、错误页和业务数据流。
- 保留 Rbac.Api 生成的动态导航树及现有退出登录逻辑。
- 文档、问题反馈等尚无真实地址的辅助入口以“暂未开放”状态呈现，不执行无效跳转。
- 不进行与共享应用壳无关的页面重构。

## 组件结构

- `AppLayout.vue`：负责页面纵向布局、引导错误、主内容和 Header/Footer 组合。
- `AppHeader.vue`：负责品牌、动态导航、分组菜单、当前路由状态、用户头像和退出操作。
- `AppFooter.vue`：负责品牌说明、权限过滤后的业务快捷入口、辅助入口、内部使用提示和版权信息。

Header 和 Footer 通过属性接收导航及用户信息，并通过事件把退出操作交还布局层处理；它们不直接操作认证引导流程。

## Header 视觉与交互

- 头部占据全视口宽度，60px 高，吸附在顶部，使用半透明 `#121212` 背景、模糊效果和 `#2c2c2c` 底边框。
- 内容区最大宽度 1320px，水平留白 28px。
- 品牌使用双色字标：`Skill` 为浅色，`ify` 为主题青色；副标题保持低对比度。
- 导航沿用动态菜单。叶子菜单直接显示；分组菜单保留下拉能力，并补齐键盘聚焦、悬停及当前路由高亮。
- 用户区展示由用户名生成的两字符头像、用户名及退出按钮。头像不依赖外部图片。
- 窄窗口允许导航换行或横向容纳，避免与用户区重叠；桌面布局仍为主要目标。

## Footer 视觉与内容

- 页脚位于共享壳底部，使用顶部细分隔线和比正文更低的文字对比度。
- 上层包含品牌说明、当前用户可见的动态业务快捷入口，以及“使用文档”“问题反馈”等辅助入口。
- 尚未接入的辅助入口显示“暂未开放”，使用按钮语义且不改变当前地址，便于后续集中替换为真实 URL。
- 下层包含版权信息和“仅供内部使用”提示。
- 页面采用最小视口高度的纵向 flex 布局：短页面时页脚贴近视口底部，长页面时自然排在内容之后。

## 数据与错误处理

- 动态导航继续来自 `menu.navTree`，不新增静态业务入口，不绕过 RBAC 可见性。
- 用户名继续来自认证 store；空用户名时头像使用通用占位字符。
- `menu.bootstrapError` 仍由布局显示，Header/Footer 不吞掉或改变引导错误。
- 退出事件继续调用现有 `logout(router)`。

## 测试与验收

- 先增加失败测试，覆盖头像文本生成、动态菜单渲染契约、模拟入口不跳转和共享壳 Header/Main/Footer 结构。
- 验证当前路由高亮及分组菜单结构未破坏动态路由行为。
- 运行前端完整 Vitest、Vue 类型检查和 Vite 生产构建。
- 在可运行环境中检查桌面宽度下的 Header/Footer 视觉，以及窄窗口不重叠。
